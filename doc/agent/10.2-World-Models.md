## §0 真问题：为什么今天的 LLM Agent 撞墙了？

把镜头拉到 2026 年中——这是一个所有大厂都在喊 "Agent" 的年份，也是一个 Agent 实际部署故事大量翻车的年份：

- **长程规划崩塌**：一个写代码的 Agent 能漂亮地完成单步重构，把它放到 30 步的 epic 上，错误率从 5% 累乘到 80%（参见 9.2 关于 Zylos 误差放大的数据）。
- **物理直觉缺席**：让 LLM 推理"杯子从桌边推下会怎样"，它知道"会掉"，但说不出杯子落地需要多少时间、碎片散布形状——它是从文本中**记住**而不是从世界中**学到**。
- **反事实失灵**：问 Agent "如果我没批准上一步会怎样"，它要么瞎编，要么基于幻觉续写。
- **昂贵动作不可学习**：机器人抓取、UI 操作、科学实验，每次失败都付出真实代价。今天的 RL 在游戏/数学/代码上有用，是因为环境**便宜**；放到现实世界，奖励信号成本暴涨，sample inefficiency 立刻致命（[Sherry Yang, arXiv:2602.00785, 2026](https://arxiv.org/abs/2602.00785)）。

这五个症状指向同一个空缺：**Agent 缺少一个能预测"如果我做了 X，世界会变成 Y"的内部模型**。这就是 World Model 要解决的问题。

> **第一性原理判句**：
> Agent 智能 = 感知 + **预测** + 规划 + 行动。
> 没有预测能力的 Agent 只能反应（reactive），不能远见（deliberative）。
> 而预测能力的载体，就是 World Model。

## §1 第一性原理：World Model 到底是什么？

### 1.1 形式化定义

World Model 是 Agent 内部的**动力学预测器**。对一个 POMDP 环境 (𝒳, 𝒜, Ω, T, O, R, γ)，World Model 至少需要学会四个操作（参考 [Agentic World Modeling Survey, arXiv:2604.22748, 2026](https://arxiv.org/abs/2604.22748)）：

| 操作 | 类型签名 | 说明 |
|---|---|---|
| State Inference | `o → s` | 从观察推断隐状态 |
| Forward Dynamics | `(s, a) → s'` | 给定状态和动作，预测下一状态 |
| Observation Decoding | `s → o` | 从状态生成观察（可选，看应用） |
| Inverse Dynamics | `(s, s') → a` | 给定状态转移，推断造成它的动作 |

注意：**预测可以在 pixel 空间，也可以在 latent 空间**。这是 World Model 设计的第一个分岔点，下一节展开。

### 1.2 为什么 LLM 不是 World Model？

LLM 是 **token 自回归预测器**：`p(token_{t+1} | token_{1..t})`。它的强项：
- 大规模文本拟合 → 知道事实
- in-context 模式匹配 → 知道格式

它的本质短板：
- **没有状态**：每次预测都是基于上下文的"重新猜"，没有持久的世界表示
- **没有动作语义**：动词 `pour_water` 对它只是 token，不绑定状态转移
- **没有因果**：相关 vs 因果在文本统计上区分不出来

CoT 不能补这个洞——你让模型"想一想"，它还是在 token 空间里折腾，没有进入一个可执行的状态空间（参见 [Oxford AIGI: CoT is not Explainability, 2025](https://aigi.ox.ac.uk/blog/chain-of-thought-is-not-explainability/) 在 9.4 中的讨论延伸）。

> **判句**：LLM 是 World Knowledge 的容器，不是 World Model 的实现。

### 1.3 World Model 的三个生效路径

World Model 一旦存在，可以在 Agent loop 中以三种方式提供价值：

1. **Planning via Simulation**：在 latent 中跑前向 rollout，挑最优动作，再执行（Dreamer/MuZero/V-JEPA 2-AC 的范式）。
2. **Counterfactual Reasoning**：模拟 "如果当时选了另一动作会怎样"，支持解释与反事实分析（见 §4 与 9.4 的联动）。
3. **Reward Shaping / Data Augmentation**：用 World Model 生成 imagined trajectories 训练策略网络（model-based RL 经典做法）。

这三条路径共同回答了 §0 的痛点：**长程靠 simulation 拼接、物理靠 latent dynamics、反事实靠 imagined rollout、昂贵动作靠 model-based RL 降本**。

## §2 JEPA：LeCun 的非自回归路线

### 2.1 JEPA 的核心 idea

Joint-Embedding Predictive Architecture (JEPA) 由 Yann LeCun 推动，与主流自回归生成模型形成鲜明对比：

| 维度 | 自回归（GPT/Sora）| JEPA |
|---|---|---|
| 预测目标 | 像素 / token | **embedding** |
| 损失 | 重建（MSE / CE）| latent space 中的距离 |
| 训练 | 大量监督生成 | self-supervised mask-denoising |
| 哲学 | 用像素细节告诉你未来 | 用语义骨架告诉你未来 |
| 失败模式 | 浪费算力在不可预测的细节（毛发、噪声） | 表示坍塌（collapse），需要 EMA teacher 防止 |

JEPA 的赌注：**世界的可预测部分在抽象表示中，像素细节的预测既无意义又浪费 FLOPs**。

### 2.2 V-JEPA 2：把 JEPA 推到能规划的程度

[V-JEPA 2 (Meta, arXiv:2506.09985, 2025)](https://arxiv.org/abs/2506.09985) 是这条路线第一个真正"能控制机械臂"的世界模型：

**两阶段训练**：
- **Stage 1（无动作预训练）**：在 VideoMix22M（百万小时级互联网视频）上做 mask-denoising，learn 通用动力学先验。ViT-g 1B 参数编码器 + ViT-small 预测器，使用 3D-RoPE。
- **Stage 2（动作条件后训练）**：在 **不到 62 小时的 Droid 机器人视频** 上做条件预测，得到 V-JEPA 2-AC（300M 预测器）。**编码器冻结**，只学动作 grounding。

**部署效果（Franka Emika Panda 机械臂，零样本，10 试验/任务）**：

| 任务 | 基线 Octo | V-JEPA 2-AC |
|---|---|---|
| Reach（触达） | 100% | 100% |
| Grasp Cup / Box | 15% / 0% | **65% / 25%** |
| Pick-&-Place Cup / Box | 15% / 10% | **80% / 65%** |

**规划方法**：Cross-Entropy Method (CEM) 在 latent 空间最小化与目标表示的 L1 距离，每步 16 秒规划 800 个候选——比 Cosmos 快 ~15×。

**两条经验**：
1. **编码器复用**：互联网视频学到的通用表征 + 极少量任务交互数据，足够支撑现实世界规划。
2. **latent space 规划**：不需要解码到像素就能做 receding-horizon planning，速度提升一个量级。

### 2.3 JEPA + 价值函数：进入 Goal-Conditioned 规划

[Value-Guided Action Planning with JEPA (arXiv:2601.00844, LeCun et al., 2025)](https://arxiv.org/abs/2601.00844) 进一步把规划做扎实：在训练时**塑造表示空间**，使 goal-conditioned value 等价于 embedding 之间的距离（或拟距离）。

这意味着：
- 找最优动作 = 找让 embedding 距离目标最近的动作
- 距离即代价，**几何即规划**

这个方向是 World Models 走向 Agent 主流的关键桥梁——它把"如何选动作"从需要奖励函数的 RL 问题，变成了几何最近邻问题。

## §3 Genie 3：可交互世界模型的新形态

如果 V-JEPA 2 是"latent 规划派"，那么 [Genie 3 (DeepMind, 2025)](https://deepmind.google/blog/genie-3-a-new-frontier-for-world-models/) 代表"pixel 仿真派"的极致：

**核心规格**：
- **720p @ 24fps 实时交互**——人类可即时操控
- **几分钟级 consistency** —— 视觉记忆约 1 分钟
- **Promptable World Events**：文本指令在仿真中途修改环境（换天气、加角色、做反事实）
- 不依赖显式 3D 表示（NeRF/Gaussian Splatting），一致性从模型涌现

**Agent 训练应用**：
- DeepMind 用其训练 **SIMA 3D 通用 Agent**——给定目标，让 Agent 在 Genie 3 worlds 里靠导航动作完成任务
- "无限课程"（unlimited curricula）：传统 RL 限于固定环境，World Model 提供组合爆炸级的训练场

**局限（DeepMind 自报）**：
- 动作空间仍狭窄
- 多 Agent 交互建模困难
- 不能精确复刻真实地理位置
- 文本渲染靠 prompt 含文本才好
- 交互时长还在"分钟"而非"小时"级

**两派对比**：

| 维度 | Latent 派 (V-JEPA 2) | Pixel 派 (Genie 3 / Sora) |
|---|---|---|
| 适合任务 | 机器人控制 / 短程规划 | Agent 训练沙盒 / 反事实模拟 |
| 计算成本 | 低（不解码） | 高（生成像素流） |
| 可视化 | 弱 | 强 |
| 与 LLM 集成 | 通过 latent → LLM token | 通过视频 → VLM 感知 |
| 当前规模 | 1B 参数级 | 推测十亿到百亿级 |

> **架构师视角**：两派不是对立，而是 **inner loop（latent 快速规划）** 与 **outer loop（pixel 训练课程）** 的关系。生产中很可能两者并存。

## §4 World Model 在 Agent Loop 中怎么用？

回到工程化视角，World Model 如何接入一个真实 Agent？以下是当前最有可能落地的几种模式：

### 4.1 模式 A：Plan-Then-Execute 升级版

```
Goal → [LLM Planner] → 候选动作序列
                          ↓
              [World Model Simulator]
                          ↓
              latent rollout × N candidates
                          ↓
                  Value/Cost Ranker
                          ↓
              选最佳 → 执行第一步
                          ↓
                  观察实际 → 重规划
```

**关键变化**：LLM 不再是唯一决策者，**World Model 在 latent 空间快速否决坏方案**。对 9.2 中 L3/L4 级 Agent 尤其关键——它给 Approver 模式提供了"为什么这个方案被推荐"的物理依据，不只是"LLM 觉得行"。

### 4.2 模式 B：Counterfactual 解释引擎

接 9.4 可解释性 UX：当用户问 "为什么 Agent 选 A 而不是 B"，让 World Model 跑 B 的 rollout，把模拟结果作为反事实证据呈现。

```python
def explain_decision(history, chosen_action, alternative):
    actual_trajectory = world_model.rollout(history, chosen_action, steps=10)
    counterfactual = world_model.rollout(history, alternative, steps=10)
    return compare(actual_trajectory, counterfactual)
    # → "如果选 B，第 4 步会触发约束 X，导致回滚"
```

### 4.3 模式 C：Model-Based RL 替代昂贵 trial-and-error

UI Agent / 浏览器 Agent / 机器人 Agent 每次真实点击/抓取都有代价。World Model 提供 imagined rollout 让策略网络在内部"做梦训练"（参考 Dreamer V3 范式）。

### 4.4 模式 D：作为推理时 verifier

LLM 生成多个 plan → World Model 模拟每个 plan 的中间状态 → 用 simulator 输出做 verifier，过滤明显违反约束/物理的方案。这与 [1.5 推理模型范式](./1.5-推理模型范式.md) 中 best-of-N + verifier 模式天然结合。

## §5 能力分级：L1 → L2 → L3

借用 [Agentic World Modeling Survey](https://arxiv.org/abs/2604.22748) 的能力分级，给架构师一个可对标的发展坐标：

| 级别 | 能力 | 边界条件 | 现状（2026 中）|
|---|---|---|---|
| **L1 Predictor** | 局部 Markov 一步预测 | 满足 4 个操作（state/forward/observation/inverse） | 大量系统已达成（Dreamer/IRIS/V-JEPA） |
| **L2 Simulator** | 多步可决策 rollout | 长程一致性 + 干预敏感 + 约束守恒 | V-JEPA 2-AC、Genie 3 部分达成（几分钟级） |
| **L3 Evolver** | 证据驱动的模型修订 | 自主诊断失败 + 持久资产更新 + 受控验证 | 几乎没有系统达到（最前沿研究目标） |

**法则维度（Law Regimes）**——每个级别还要分场景标注：
1. **Physical**（接触力学）—— 机器人 / VR
2. **Digital**（程序语义、UI 状态机）—— 浏览器 Agent / OS Agent（与 [10.4](./10.4-OS-level-Agent.md) 紧密相关）
3. **Social**（信念、目标、规范）—— 多 Agent 协作
4. **Scientific**（潜在因果机制）—— 科学实验 Agent

> **判句**：今天的 LLM Agent 大致处于 **L0 → L1**（只有零散预测），World Model 工程的目标是把它推到 **L2**，把 SOTA 推到 **L3**。

## §6 关键代码骨架：Latent 规划循环

下面是一个最小可参考的 latent planning loop 骨架（伪 Python，PyTorch 风格）：

```python
class WorldModelAgent:
    def __init__(self, encoder, predictor, policy_proposer, value_fn):
        self.encoder = encoder            # frozen, large
        self.predictor = predictor        # small, action-conditioned
        self.policy = policy_proposer     # could be LLM-generated candidates
        self.value = value_fn             # goal-conditioned distance

    def plan(self, obs, goal, horizon=8, samples=512):
        z0 = self.encoder(obs)
        z_goal = self.encoder(goal)

        # 1. 用 LLM 或 CEM 提议候选动作序列
        candidates = self.policy.propose(z0, z_goal, n=samples)  # [N, H, A]

        # 2. 在 latent 中 rollout
        best_seq, best_score = None, float("inf")
        for action_seq in candidates:
            z = z0
            for a in action_seq:
                z = self.predictor(z, a)
            score = self.value(z, z_goal)
            if score < best_score:
                best_score, best_seq = score, action_seq

        # 3. 只执行第一步，下一帧重新规划（receding horizon）
        return best_seq[0]

    def act(self, env, goal):
        obs = env.reset()
        while not done(obs, goal):
            a = self.plan(obs, goal)
            obs = env.step(a)
```

**生产化几个关键点**：
- **encoder 不微调**——保持其通用表示能力，模仿 V-JEPA 2-AC
- **predictor 量化压缩**——latent rollout 是性能瓶颈，需要 KV cache + 量化
- **candidate proposal 可分层**：LLM 高层次提议 → CEM 局部细化
- **safety guard**：rollout 时检测约束违反，提前剪枝（呼应 [6.2 Guardrails](./6.2-Guardrails三层护栏.md)）

## §7 反模式（不要做这些）

| 反模式 | 后果 | 替代 |
|---|---|---|
| 把 LLM 直接当 World Model | 长程崩塌、物理乱写 | 引入显式 latent dynamics |
| 在 pixel 空间做规划 | GPU 烧到天上、速度 1FPS | latent 空间 + 必要时解码 |
| World Model + 策略一锅端训 | 表示坍塌 / 训不稳 | 模块化、冻结编码器（V-JEPA 2 做法） |
| 不验证 simulator 与真实的 gap | "模拟成功，实际翻车" | sim-to-real gap 监控 + 在线校准 |
| World Model 用于解释但不做 calibration | 给用户假信心 | 反事实 + 置信度区间 |
| 把 L1 当 L2 用 | 简单任务行，复杂全崩 | 显式标注能力级别，门控自主度（接 9.2）|
| 忽略 Digital/Social 法则 | 只关注 Physical 场景，UI/多 Agent 翻车 | 按 Law Regime 分别建模 |

## §8 S0 → S5 落地路线图

写给"明天就要做"的工程团队：

### S0：现状（多数 Agent）
- 没有显式 World Model
- LLM + tools，全靠 trial-and-error
- 长程任务失败率 > 50%

### S1：Verifier 引入（1-2 个月）
- LLM 仍生成 plan
- 在关键步骤接入轻量预测器（规则 + 小模型）做可行性校验
- 立即收益：明显 hallucination 的方案被拦截

### S2：Latent Simulator MVP（3-6 个月）
- 在你的 domain 上训练一个 latent dynamics 模型（可在通用 backbone 上微调）
- Plan-Then-Simulate：所有候选先 latent rollout 排序
- 减少 30-50% 真实执行错误（参考 V-JEPA 2 抓取从 15%→80%）

### S3：Counterfactual UX 上线（6-9 个月）
- World Model 提供解释能力（4.2 模式 B）
- 与 9.4 解释 UX 联动，给用户反事实证据

### S4：Model-Based RL 训练循环（9-12 个月）
- 用 simulator 生成 imagined data 训练策略
- 真实交互成本下降一个数量级

### S5：L3 Evolver 试点（12+ 个月）
- Agent 能自主发现 World Model 失效
- 触发数据采集 → 模型修订 → 受控验证
- 当前 SOTA 研究方向，工业界鲜有落地

## §9 一页总结

**判句**：LLM Agent 的下一次跃迁不在更大的 token 模型，而在能预测世界的 latent dynamics。

**三件事必须记住**：
1. **JEPA 系（latent 派）解决规划，Genie 系（pixel 派）解决训练课程**——两派将共存而非择一。
2. **Encoder 通用、Predictor 轻量、Frozen 训练**是 V-JEPA 2 给出的关键工程范式。
3. **L1→L2→L3** 是衡量你团队所处位置的统一坐标——不要把 L1 系统当 L2 用。

**做对的事**：
- 把 World Model 视为 Agent 架构第一公民，不是"加个外挂"
- 先做 latent verifier（S1），再做 simulator（S2）
- 训练时坚持模块化、frozen encoder
- 评估时 always 检查 sim-to-real gap

**警惕**：
- "LLM = World Model" 的混淆
- pixel-space planning 的算力陷阱
- 没有 calibration 的反事实解释
