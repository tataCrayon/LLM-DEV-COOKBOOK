## 三、P1 强烈建议（10 篇）

### 1.4 Function Calling / Tool Use → P1
- MCP 成为事实标准（97M downloads），原文可能低估了其生态规模
- Anthropic Advanced Tool Use（2025-11）：Tool Search（85% token 减少）、Programmatic Tool Calling

- 工具可靠性基准数据：Anthropic 8.4/10 vs Google 7.9/10 vs OpenAI 6.3/10

### 1.6 结构化输出工程 → P1
- XGrammar-2（2026-05）：100x 提速，成为 vLLM/SGLang/TensorRT-LLM 默认后端
- Pydantic AI 成为正式 Agent 框架
- BAML Schema-Aligned Parsing（SAP）处理 CoT + JSON 混合输出
- Instructor 12K+ stars / 3M+ monthly PyPI downloads
- 三大厂商原生结构化输出 API 成熟：OpenAI `.parse()` / Anthropic `output_config` / Google `response_schema`

### 2.2 Harness Engineering → P1
- Martin Fowler 文章正式发布（2026）：Guides/Sensors 分类法 + 三类调节（可维护性/架构适应度/行为）
- "Harnessability" 概念：不是所有代码库都适合 harness
- Harness Templates：预打包的 guide+sensor 套件

### 2.3 Agent 隔离技术 → P1（也是 10.1）
- Agent sandbox 生态爆发：100+ 方案，每个主流 coding agent 都内置沙箱
- Docker 发布官方 sandbox 产品 (`sbx`)
- Firecracker 主导托管层：~125ms 启动、<5MiB 开销、150 VMs/s/host
- WASM/WASI 作为轻量隔离层兴起
- Recursive Self-Improvement：Anthropic 数据显示 80% 合并代码由 Claude 编写、52x 代码优化加速
- CSA 发布"递归自改进安全影响"报告

### 3.7 Multi-Agent 可靠性 → P1
- CP-WBFT（arXiv:2511.10400，AAAI 发表）：基于 LLM 反思能力的 Byzantine 容错共识机制
- 在 85.7% 故障率下仍有效——超越传统 BFT
- 和 v1.0 的 MAS-FIRE 数据形成互补

### 5.2 RAG 工程深水区 → P1
- Microsoft GraphRAG 2.0：实体抽取 78% → 92%，多跳问题 +40%，幻觉 -25%
- Agentic RAG 成为主流范式（Agent 自主决定检索策略 + 自纠正）
- Corrective RAG (CRAG) pattern
- 2026 RAG 分类法已扩展到 20+ 种模式

### 5.3 长期记忆架构 → P1
- Hindsight 作为 Letta 替代方案兴起（多策略并行检索 + 交叉编码器 reranking，LongMemEval 94.6%）
- "Memory in the Age of AI Agents"（arXiv:2512.13564，102 页综述）——定义性分类法
- 两大范式分化：Agent 自编辑（Letta）vs 被动系统抽取（Hindsight）

### 6.1 Prompt Injection 攻防 → P1
- 新增第 3-5 类攻击：Memory Poisoning / Tool-Chain Exploitation / Multi-Step Injection Chains
- 2026 防御共识：5 层纵深（输入清洗 / 权限分离 / 输出验证 / HITL / 监控异常检测）
- Prompt Injection 仍为 OWASP #1

### 6.5 Agent Identity 与供应链安全 → P1
- W3C Agent Identity Registry CG（2026-04 启动）——DID method、Agent credential format、Trust negotiation protocol
- IETF draft-klrc-aiagent-auth：9 层 AIMS 架构
- 静态 API key 被明确定义为"反模式"

### 7.3 可观测性平台 → P1
- LangSmith Engine（自动调试 Agent，打开 PR 修 bug）
- SmithDB（Rust + Apache DataFusion，15x 提速）
- Braintrust Loop/Brainstore
- MLflow 3.10（OTel 原生 + 60 框架集成）
- Galileo AI（可在 tool 执行前干预的 guardrail）
- Langfuse 社区崛起
